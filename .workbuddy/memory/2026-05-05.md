## 卡住问题进一步修复
- 用户反馈：点3次下个月仍然卡住
- 排查发现两个累积性问题：
  1. `initDragSelection` 每次 loadSchedule 都重复添加 mousedown 监听器，不清理旧的

  - 给 `initDragSelection` 加 `_dragSelectionInitialized` 标记，只初始化一次
  - 给 `loadSchedule` 加 `_isLoading` 锁，防止并发执行

## 月视图导航和年份显示修复
- 用户反馈两个问题：
  1. 月视图点击「下个月」只能点3次，之后继续点没反应
  2. 右上角日期有时显示年份，有时不显示
- **问题1根因**：`monthFirstMonday()` 返回的日期可能在上个月（如7月第一个周一是6月29日），用 `getMonth()` 计算导航会出错
- **修复**：新增 `getRepresentedMonth()` 辅助函数，通过判断该周周日是否跨到下个月来推断当前周「代表」的月份
- **问题2根因**：原逻辑只在 `startYear !== endYear` 时显示年份，跨到未来同一年份时不显示
- **修复**：增加 `currentYear` 判断，只要日期范围不在当前年份就显示年份
- 修改文件：public/js/schedule.js
- 提交 commit 25584ed 并推送到 GitHub main
