# 2026-04-08

## 项目概览：crewboard（神马排班）

首次代码了解，完整分析如下：

### 技术栈
- **后端**：Node.js + Express + SQLite（better-sqlite3）
- **前端**：纯 Vanilla JS SPA，无构建工具，无框架
- **部署**：PM2 + Nginx 反向代理，运行于 resource.skandstudio.com

### 核心功能模块
1. **资源排程**（schedule.js）：周视图/月视图，可视化排程日历，支持批量多人排程
2. **工时表**（timesheets.js）：按人员按周填报，支持批量保存
3. **报表**（reports.js）：利用率报表 + 项目报表，Excel 导出
4. **人员管理**（manage.js）：Resources CRUD，项目/客户管理，存档功能
5. **企业管理**（enterprise.js）：多租户企业系统，成员权限管理，企业 Webhook 通知
6. **认证**（auth.js）：手机/邮箱登录注册，scrypt 密码加密，7天 Session Token

### 数据库（SQLite）11张表
- resources, clients, projects, bookings, timesheets, leave_entries
- enterprises, users, join_requests, sessions, invitations

### 文件结构
- `server.js` - 入口，Express 初始化
- `db/schema.js` - 数据库 schema + 迁移 + 演示数据
- `db/holidays.js` - 中国节假日数据（2025-2026），自动更新
- `routes/api.js` - 核心业务 API（24KB）
- `routes/auth.js` - 认证 & 企业管理 API（19KB）
- `routes/webhook.js` - 钉钉/企微/飞书 Webhook 推送
- `public/index.html` - SPA 单页，8个页面视图
- `public/css/style.css` - CSS（44KB），支持 12 种主题色
- `public/js/core.js` - 全局 state、API helper、导航
- `public/js/schedule.js` - 排程页面（42KB，最大）
- `public/js/manage.js` - 人员/项目/客户管理（37KB）
- `public/js/enterprise.js` - 企业/设置/账号（27KB）
- `public/js/reports.js` - 报表（8KB）
- `public/js/timesheets.js` - 工时表（8KB）

### 特色
- 中国本土化：节假日系统、人民币计费、钉钉/企微/飞书集成
- 多租户数据隔离（enterprise_id）
- 无前端框架，极简 4 个 npm 依赖
- 支持 SEED_DEMO=1 一键生成演示数据

- **实现**：在 `schedule.js` 中添加 `initResizeBooking()` 函数
- **功能**：鼠标悬停在预定块右侧边缘，出现可拖拽的 `resize-handle`，拖动可缩短或延长预定期限
- **交互**：拖拽时显示视觉引导线，释放后确认操作，支持向左拖动（缩短/移动）和向右拖动（延长）
- **CSS**：添加 `.resize-handle` 样式和 `.resizing` 状态类，修复了 `overflow: hidden` 导致手柄不可见的问题
- **权限**：检查 `canBookForResource()` 确保用户有编辑权限
- **修复**：`resize-handle` 的 `mousedown` 事件会临时禁用预定块的 `pointer-events`，防止点击编辑功能被干扰
- **增强**：提高resize handle的可见性（opacity 0.5→1 on hover），增加宽度（12px→16px），扩展点击区域（right: 0→-4px）

## 2026-04-08 修复记录

### 资源排程页面 resize handle 无效问题
- **问题**：鼠标悬停在已预定的时间末端，resize handle 不可见或无法点击
- **根因分析**：
  1. `.booking-block` 设置了 `overflow: hidden`，裁剪了 resize handle
  2. Resize handle 默认 `opacity: 0`，虽然可点击但完全不可见
  3. 原始代码只支持向右拖动（延长），不支持向左拖动（缩短）
- **修复措施**：
  1. CSS：将 `.booking-block` 的 `overflow: hidden` 改为 `overflow: visible`
  2. CSS：提高 resize handle 默认可见性（opacity 0→0.6），悬停时变为1
  3. CSS：增加 resize handle 宽度（8px→16px），向右扩展4px（right: -4px）
  4. JS：修复 `initResizeBooking` 中 `endIndex` 计算逻辑，移除 `Math.max`，支持左右拖动
  5. JS：完善 `handleMouseUp` 逻辑，支持缩短（向左拖动时移动预定）和延长（向右拖动时创建新预定）
- **测试结果**：功能恢复正常，用户可鼠标悬停在预定块末端，点击并拖动调整预定时间

### 周末预定无法显示问题
- **问题**：周末的预定无法在排程页面中显示
- **根因分析**：
  1. 在 `routes/api.js` 的预定创建逻辑中，有代码明确跳过周末（`if (day !== 0 && day !== 6)`）
  2. `schedule.js` 中的 `countWorkDays` 函数只计算工作日，导致周末预定总小时数计算错误
- **修复措施**：
  1. 移除 `routes/api.js` 中跳过周末的逻辑，允许在周末创建预定
  2. 将 `schedule.js` 中的 `countWorkDays` 函数重命名为 `countAllDays`，修改为计算所有天数（包括周末）
  3. 更新预定总计显示文本，从"个工作日"改为"天"
- **影响**：现在用户可以在周末创建预定，预定会正确显示在排程页面中

### 月视图缺少resize handle功能问题
- **问题**：月视图中的预定块没有resize handle，无法进行拖动调整
- **根因分析**：
  1. 月视图预定块使用 `.m-booking` 类，而不是周视图的 `.booking-block` 类
  2. HTML生成代码中没有为月视图预定块添加resize handle元素
  3. 事件监听器只监听 `.booking-block .resize-handle`，不监听 `.m-booking .resize-handle`
- **修复措施**：
  1. 修改 `buildMonthResourceRow` 函数，为预定块添加 `data-resource-id` 属性和resize handle元素
  2. 扩展事件监听器选择器，支持 `.m-booking .resize-handle` 和 `.m-booking` 元素
  3. 为 `.m-booking` 添加CSS样式：`overflow: visible` 和 `position: relative`
  4. 为月视图resize handle添加专门的CSS样式，调整尺寸适应月视图较小的预定块
- **影响**：现在月视图中的预定块也显示紫色的resize handle，支持点击拖动调整
