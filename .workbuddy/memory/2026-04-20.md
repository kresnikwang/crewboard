## 修复编辑 booking 结束日期不生效问题
- 问题：编辑现有 booking 时修改结束日期，保存后没有效果
- 原因：`saveBooking` 函数在编辑时只更新了 `date` 字段，没有处理 `end_date`
- 解决：
  - 检测日期范围是否发生变化（比较 startDate 和 endDate）
  - 如果日期未变，执行原来的 PUT 更新
  - 如果日期变化，先 DELETE 原 booking，然后按新日期范围创建新的 booking
  - 同样处理了 leave 冲突检查和连续日期分段逻辑
- 修改文件: public/js/schedule.js

## 修复拖拽缩短 booking 跨周边界问题
- 问题：从右往左缩短排程时，每周的第一天总是删不掉
- 原因：缩短逻辑使用 `dates.indexOf(b.date)` 判断要删除的 booking，但 `dates` 只包含当前视图的日期
- 解决：
  - 改用日期字符串比较（`b.date > hoverDate && b.date <= endDate`）
  - 在 `initResizeBooking` 中添加逻辑找出连续段的结束日期 `endDate`
  - 在 `initResizeBookingLeft` 中使用 `startDate` 和 `hoverDate` 进行日期比较
- 修改文件: public/js/schedule.js

## 修复跨周 booking 的 resize handle 问题
- 问题：红框（booking）无法拖到第一格，单日 booking 的右侧边缘无法点中
- 原因：`detectSpans` 只根据当前视图内的日期判断 span-s/m/e，导致跨周边界的 booking 计算错误
  - 如果 span 开始于视图外，视图内的第一个 booking 会被误认为 span-s（有左调整手柄但无右调整手柄）
  - 如果 span 结束于视图外，视图内的最后一个 booking 会被误认为 span-e（有右调整手柄但无左调整手柄）
- 解决：
  - 重写 `detectSpans`，使用 `_allBookings` 全局数据来计算完整的 span 范围（不限于当前视图）
  - 计算每个 booking 所在 span 的真正开始日期和结束日期（可能跨越视图边界）
  - 使用完整 span 信息判断 hasPrev/hasNext，正确分配 span-s/m-e 类名
- 修改文件: public/js/schedule.js

## 修复缩短 booking 时忽略 split_after 标记的问题
- 问题：用户在三天 booking 的第一天后切了一刀（split_after=1），然后从第一天右侧向左缩短时，三天的 booking 都被删除了

## 修复 split_after booking 左侧不显示 resize handle 的问题
- 问题：王强 4 月 22 日的 booking（有 split_after=1）左侧不显示分隔 UI，也无法从左边缩短
- 原因：`detectSpans` 中 `isSplitPoint` 优先级最高，直接将 cls 设为 `span-e`，覆盖了 `isAfterSplit` 的判断。`span-e` 没有 `hasBorderLeft`，所以左侧没有 resize handle
- 根本原因：`split_after` 只影响右侧连接（不应连接到下一天），不应影响左侧连接。但旧逻辑把 `isSplitPoint` 当作"强制 span-e"，导致左侧也被截断
- 解决：重写 span 类名分配逻辑，使用 `effectiveHasNext` 和 `effectiveHasPrev`：
  - `effectiveHasNext = hasNext && !isSplitPoint`（split_after 阻止右侧连接）
  - `effectiveHasPrev = hasPrev && !isAfterSplit`（前面有 split 阻止左侧连接）
  - 根据 effective 值决定 cls，不再强制覆盖
- 修改文件: public/js/schedule.js
