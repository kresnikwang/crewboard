## 修复排程页面日期表头滚动问题 #2（真正根因）
- 提交: 1467320 fix(schedule): 修复双滚动容器导致sticky表头失效
- 背景：第一轮修复（border-collapse → separate）后问题仍存在
- 真正根因：**双层滚动容器竞争**
  - `.main-content` 有 `overflow-y: auto`（外层滚动容器）
  - `.schedule-grid` 有 `flex: 1; overflow: auto`（内层滚动容器）
  - flex column 中缺少 `min-height: 0`，导致 flex 元素无法限制内容高度
  - 内容溢出不发生在 `.schedule-grid` 内部，而是传播到 `.main-content`
  - sticky th 的相对容器是 `.schedule-grid`，但实际滚动发生在 `.main-content`
  - 结果：th 随 `.main-content` 一起被滚走

### 修复
| 文件 | 改动 |
|------|------|
| `style.css:40` | `.page`: 添加 `min-height: 0` |
| `style.css:80` | `.schedule-grid`: 添加 `min-height: 0` |
| `style.css:263` | `.timesheet-container`: 添加 `min-height: 0` |
| `style.css:282` | `.report-container`: 添加 `min-height: 0` |
| `style.css:381,1031` | `#clients-projects-container`: 添加 `min-height: 0` |
